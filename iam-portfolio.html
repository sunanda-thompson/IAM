IAM Fundamentals

üîê RBAC
Role-Based Access Control
Permissions are assigned to roles, not individuals. Users gain access by being assigned to roles that carry the necessary permissions ‚Äî reducing admin overhead and enforcing least privilege.
Least Privilege Role Assignment Access Review
üé≠ ABAC
Attribute-Based Access Control
Access decisions are made using attributes of users, resources, and the environment. More granular than RBAC ‚Äî a user's department, clearance level, location, and time of day can all influence access.
Policy Engine Dynamic Access Context-Aware
üè¢ AD / LDAP
Directory Services
Microsoft Active Directory and LDAP-based directories are the backbone of enterprise IAM. Managing OUs, Groups, GPOs, and Service Accounts is a daily DBA/IAM task in most organizations.
Active Directory GPO OU Structure LDAP
üõ°Ô∏è PAM
Privileged Access Management
Securing, controlling, and auditing access for privileged accounts (admins, root, service accounts). PAM tools vault credentials, enforce MFA, session recording, and just-in-time access.
Just-in-Time Credential Vault Session Audit
üîë SSO / MFA
Authentication & Federation
Single Sign-On (SSO) reduces friction while maintaining security. MFA adds layered verification. Protocols like SAML 2.0, OAuth 2.0, and OIDC enable federated identity across cloud services.
SAML 2.0 OAuth 2.0 OIDC MFA
üö´ ZT / PoLP
Zero Trust & Least Privilege
Never trust, always verify. Zero Trust assumes breach and validates every request regardless of network location. Principle of Least Privilege ensures users have only the minimum access required for their role.
Zero Trust Microsegmentation Continuous Auth
